## Descripción de los programas desarrollados

En esta actividad desarrollé dos programas en Python que resuelven el mismo problema: registrar y mostrar la información básica de una mascota, incluyendo su nombre, especie y edad.

El primer programa fue elaborado utilizando el enfoque de Programación Tradicional. Para ello empleé funciones, variables y entrada de datos por teclado. El programa permite registrar la información de una mascota y posteriormente mostrarla de manera organizada en pantalla. En esta solución no se utilizaron clases ni objetos, siguiendo las características propias de la programación tradicional.

El segundo programa fue desarrollado mediante el enfoque de Programación Orientada a Objetos. En este caso implementé una clase llamada **Mascota**, la cual contiene atributos para almacenar la información de cada mascota y métodos para mostrar sus datos y representar un comportamiento mediante la emisión de sonidos. Además, creé objetos de la clase para demostrar cómo cada instancia puede almacenar información independiente y ejecutar sus propios métodos.

## Reflexión sobre las diferencias encontradas

Durante el desarrollo de esta actividad pude comprender de manera práctica las diferencias entre la Programación Tradicional y la Programación Orientada a Objetos.

En la Programación Tradicional, la solución se construye principalmente mediante funciones que procesan datos almacenados en variables. Considero que este enfoque es más sencillo de entender cuando se trabaja con programas pequeños, ya que la lógica se desarrolla de forma directa y estructurada. Sin embargo, a medida que aumenta la cantidad de información o funcionalidades, la organización del código puede volverse más compleja.

Por otra parte, la Programación Orientada a Objetos permite organizar mejor la información al agrupar datos y comportamientos dentro de una misma estructura denominada clase. Al trabajar con objetos, observé que es más fácil representar elementos del mundo real y mantener una relación clara entre sus características y acciones. Además, este enfoque favorece la reutilización del código y facilita la ampliación de los programas en el futuro.

En conclusión, ambos enfoques permiten resolver un mismo problema, pero la Programación Orientada a Objetos ofrece una estructura más organizada, reutilizable y escalable. Esta práctica me permitió identificar las ventajas de cada metodología y comprender la importancia de seleccionar el enfoque más adecuado según las necesidades del proyecto.
